# 🤖 Android Build Fix - SDK Configuration

## Problem
The Android build is failing with:
```
SDK location not found. Define a valid SDK location with an ANDROID_HOME 
environment variable or by setting the sdk.dir path in your project's 
local properties file
```

## Solution Options

### Option 1: Set ANDROID_HOME Environment Variable (Recommended)

1. **Find your Android SDK location**:
   - Typical location: `C:\Users\<YourUser>\AppData\Local\Android\Sdk`
   - Or check Android Studio: `File > Settings > Appearance & Behavior > System Settings > Android SDK`

2. **Set environment variable**:
   ```powershell
   # Open PowerShell as Administrator
   [System.Environment]::SetEnvironmentVariable('ANDROID_HOME', 'C:\Users\Administrator\AppData\Local\Android\Sdk', 'User')
   [System.Environment]::SetEnvironmentVariable('ANDROID_HOME', 'C:\Users\Administrator\AppData\Local\Android\Sdk', 'Machine')
   
   # Restart your terminal/IDE after
   ```

3. **Verify**:
   ```powershell
   echo $env:ANDROID_HOME
   # Should output: C:\Users\Administrator\AppData\Local\Android\Sdk
   ```

---

### Option 2: Create local.properties File

Create file: `mobile-app/android/local.properties`

```properties
## This file is automatically generated by Android Studio.
## Do not modify this file -- YOUR CHANGES WILL BE ERASED!
##
## This file must *NOT* be checked into Version Control Systems,
## as it contains information specific to your local configuration.
##
## Location of the android SDK.
## For customization when using a Version Control System, please read the
## header note.

sdk.dir=C:\\Users\\Administrator\\AppData\\Local\\Android\\Sdk
```

**Note**: Use double backslashes (`\\`) in the path!

---

### Option 3: Quick Fix Script (PowerShell)

Save this as `fix-android-sdk.ps1`:

```powershell
# Android SDK Auto-Fix Script
$ErrorActionPreference = "Stop"

Write-Host "🔍 Searching for Android SDK..." -ForegroundColor Cyan

# Common SDK locations
$possiblePaths = @(
    "$env:USERPROFILE\AppData\Local\Android\Sdk",
    "$env:LOCALAPPDATA\Android\Sdk",
    "C:\Android\Sdk",
    "C:\Users\$env:USERNAME\Android\Sdk"
)

$sdkPath = $null
foreach ($path in $possiblePaths) {
    if (Test-Path $path) {
        $sdkPath = $path
        Write-Host "✅ Found Android SDK at: $sdkPath" -ForegroundColor Green
        break
    }
}

if ($null -eq $sdkPath) {
    Write-Host "❌ Android SDK not found in common locations" -ForegroundColor Red
    Write-Host "Please install Android Studio or manually set ANDROID_HOME" -ForegroundColor Yellow
    exit 1
}

# Create local.properties
$localPropsPath = "mobile-app\android\local.properties"
$localPropsContent = "sdk.dir=$($sdkPath -replace '\\', '\\\\')"

Write-Host "📝 Creating local.properties..." -ForegroundColor Cyan
New-Item -Path $localPropsPath -ItemType File -Force | Out-Null
Set-Content -Path $localPropsPath -Value $localPropsContent

Write-Host "✅ Created: $localPropsPath" -ForegroundColor Green
Write-Host "Content: $localPropsContent" -ForegroundColor Gray

# Also set environment variable
Write-Host "🔧 Setting ANDROID_HOME environment variable..." -ForegroundColor Cyan
[System.Environment]::SetEnvironmentVariable('ANDROID_HOME', $sdkPath, 'User')

Write-Host "✅ Android SDK configured successfully!" -ForegroundColor Green
Write-Host "⚠️  Please restart your terminal/IDE" -ForegroundColor Yellow
```

Run with:
```powershell
.\fix-android-sdk.ps1
```

## Common Issues

### Issue: "SDK not found" even after setting ANDROID_HOME
**Fix**: Make sure to restart your terminal/IDE after setting environment variables.

### Issue: Path has spaces
**Fix**: Use quotes around the path:
```properties
sdk.dir=C:\\Program Files\\Android\\Sdk
```

### Issue: Permission denied
**Fix**: Run PowerShell as Administrator when setting system-wide env vars.

---

## Recommended Android SDK Setup

If you don't have Android SDK installed:

1. **Install Android Studio**: https://developer.android.com/studio
2. **Open Android Studio**
3. **Go to**: `File > Settings > Appearance & Behavior > System Settings > Android SDK`
4. **Install**:
   - Android SDK Platform (latest)
   - Android SDK Build-Tools (latest)
   - Android SDK Platform-Tools
   - Android SDK Command-line Tools
5. **Note the SDK Location** shown in the dialog
6. **Use that path** in `local.properties` or `ANDROID_HOME`
